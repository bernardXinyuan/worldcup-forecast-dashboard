# 项目总规划

## 1. 项目定位

本项目是一个面向 2026 美加墨世界杯的独立预测看板，服务于普通球迷、赛事预测爱好者、内容运营者和数据分析用户。

产品应可以直接上线使用。这里的“可上线”不只是网页能打开，而是包括稳定数据更新、预测解释、异常兜底、SEO/分享能力、监控告警、验收标准和后续维护机制。

## 2. 核心目标

### 用户目标

- 球迷可以快速了解世界杯赛程、预测、夺冠概率和热门球队走势。
- 预测爱好者可以模拟小组赛比分、查看淘汰赛路径和概率变化。
- 内容运营者可以基于预测数据生产赛事话题、榜单和分析内容。
- 数据维护者可以管理基础数据、预测数据和更新状态。

### 产品目标

- 提供一个清晰、可信、可互动的世界杯预测体验。
- 通过小组模拟器和淘汰赛路径图形成核心差异化。
- 支持社交分享和搜索访问，适合赛前预热与赛中持续访问。
- 形成可复用的赛事预测产品框架，未来可复用于欧洲杯、亚洲杯、奥运会等赛事。

### 交付目标

- 交付可运行的 Web 看板。
- 支持桌面端和移动端访问。
- 支持基础后台配置或配置文件驱动。
- 支持自动或半自动数据更新。
- 支持上线文档、验收清单和应急预案。

## 3. 目标用户与场景

### 普通球迷

在赛前或赛中查看今日比赛、球队实力、冠军概率和淘汰赛路径。

### 预测爱好者

修改比赛比分，观察小组排名、出线概率和后续对阵变化。

### 内容运营者

使用看板内容制作专题、社媒话题、赛前分析和赛后复盘。

### 数据维护者

检查赛程、结果、预测概率和更新时间是否准确。

### 技术运维

负责部署、监控、数据更新、故障处理和版本发布。

## 4. 产品边界

### MVP 必须包含

- 首页综合预测看板。
- 赛程列表和单场预测。
- 小组积分榜和出线概率。
- 小组赛比分模拟器。
- 淘汰赛路径预测图。
- 球队详情页。
- 数据更新时间和来源说明。
- 分享能力。
- 部署和验收文档。

### 第二阶段包含

- 用户投票。
- 预测排行榜。
- 赔率或市场热度对比。
- 预测命中率回顾。
- 多语言展示。
- 内容管理后台。

### 暂不包含

- 真金白银竞猜、博彩、返奖机制。
- 复杂会员系统。
- 自建专业体育数据采集团队。
- 强依赖实时直播版权内容。

## 5. 项目阶段

### P0: 规划与设计

产出 PRD、信息架构、数据模型、技术方案、部署方案和验收标准。

### P1: MVP 开发

完成预测看板核心页面、静态或半自动数据接入、模拟器、路径图和基础配置。

### P2: 内测与内容校准

邀请少量用户或团队成员试用，验证预测解释、交互体验、移动端适配和数据更新流程。

### P3: 正式上线

完成生产环境部署、监控告警、SEO、分享卡片、操作手册、版本冻结和灰度发布。

### P4: 赛中运营

根据赛程推进持续更新数据、预测、热门内容和复盘分析，处理异常和快速迭代。

## 6. 关键里程碑

| 里程碑 | 目标 | 验收标准 |
| --- | --- | --- |
| M1 需求冻结 | MVP 范围明确 | PRD、Backlog、验收清单完成 |
| M2 原型确认 | 核心体验确认 | 首页、赛程、模拟器、路径图原型确认 |
| M3 技术方案确认 | 可开发可部署 | 架构、数据源、部署路径确认 |
| M4 MVP 可运行 | 本地完整体验 | 核心页面和示例数据跑通 |
| M5 内测发布 | 小范围试用 | 访问稳定，移动端可用，关键反馈收集完成 |
| M6 正式发布 | 线上可用 | 生产部署、监控、SEO、分享和应急预案完成 |

## 7. 主要风险

- 数据源不稳定或授权不清晰。
- 预测模型解释不足，用户觉得结果不可信。
- 移动端图表和路径图展示复杂，容易影响可读性。
- 世界杯期间赛程密集，数据更新和内容运营压力大。
- 上下文窗口限制导致 agent 接力时遗漏决策。

## 8. 管理原则

- 所有关键决策必须写入本地文档。
- 每个 agent 只负责清晰边界内的任务。
- 每轮执行结束必须写交接摘要。
- 产品优先保证可上线、可维护、可验收，再追求高级预测能力。
