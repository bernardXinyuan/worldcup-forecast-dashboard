# 部署与上线交付

## 1. 交付目标

产品必须可以作为独立 Web 产品上线使用。交付范围包括应用、配置、数据、监控、SEO、分享、验收和应急预案。

## 2. 推荐部署形态

### MVP 推荐方案

- 前端：Web 应用，支持桌面端和移动端。
- 后端：轻量 API 服务，提供数据、内容配置和投票接口。
- 数据：托管数据库或云端 JSON 配置加缓存。
- 部署：云平台托管，用户通过公开 URL 访问。
- 分享：每个页面有稳定 URL，支持社交分享卡片。

示例：

```text
https://worldcup-dashboard.example.com/
https://worldcup-dashboard.example.com/matches
https://worldcup-dashboard.example.com/team/argentina
https://worldcup-dashboard.example.com/simulator?group=A
```

### 无后台兜底方案

如果时间紧，可以先使用静态站点 + JSON 数据文件：

- 预测数据定时生成 JSON。
- 内容配置放在 JSON 文件中。
- 前端定时拉取最新 JSON。
- 投票和后台延后到 P1。

## 3. 环境规划

### 本地环境

用途：开发和单元测试。

要求：

- 一条命令启动前后端。
- 提供示例数据。
- 不依赖真实生产密钥。

### 测试环境

用途：QA、内容预览和上线前验证。

要求：

- 接近生产配置。
- 支持模拟数据异常。
- 支持分享卡片预览。

### 生产环境

用途：正式用户访问。

要求：

- HTTPS。
- 访问日志。
- 错误监控。
- 数据备份。
- 可回滚。
- CDN 或静态资源缓存。

## 5. 前端上线要求

### 显示要求

- 优先适配桌面端 1440px 和移动端 390px 宽度。
- 支持常见浏览器。
- 核心页面在移动端不遮挡、不溢出。
- 路径图在移动端可横向滚动或分阶段查看。

### 性能要求

- 首页首屏加载时间小于 3 秒。
- 核心交互响应时间小于 200ms。
- 静态资源启用压缩和缓存。
- 数据异常时显示兜底状态。

### SEO 和分享要求

- 首页、球队详情、比赛详情有独立标题和描述。
- Open Graph 和 Twitter Card 信息完整。
- 重要页面可被搜索引擎抓取。
- 分享 URL 可恢复关键状态。

## 6. 发布流程

### 发布前

1. PM 确认版本范围。
2. QA 完成验收测试。
3. Content Ops 确认首页文案、热门球队和分享配置。
4. DevOps 确认部署配置和回滚方案。

### 发布中

1. 部署到生产环境。
2. 打开关键页面验证。
3. 检查移动端、桌面端和分享预览。
4. 观察错误日志和加载性能。

### 发布后

1. 收集用户反馈。
2. 监控访问、错误和数据更新时间。
3. 更新发布记录。
4. 对阻断问题执行回滚或热修。

## 7. 验收清单

### 产品验收

- 首页冠军预测榜可见。
- 今日比赛预测可见。
- 小组积分榜可用。
- 小组模拟器可修改比分并更新结果。
- 淘汰赛路径图可读。
- 球队详情页可访问。
- 数据来源和更新时间可见。
- 页面可分享。

### 技术验收

- HTTPS 可访问。
- 生产构建无明显控制台错误。
- 首屏加载时间小于 3 秒，弱网下有加载态。
- 后端 API 有错误兜底。
- 数据更新失败会告警。
- 支持回滚到上一版本。

### 内容验收

- 首页标题和文案准确。
- 热门球队配置正确。
- 免责声明可见。
- 分享卡片标题、描述、图片正确。
- 赛程和球队名称无明显错误。

## 8. 应急预案

### 数据源失败

处理：

- 使用最近一次成功数据。
- 页面显示更新时间。
- 内容或数据维护者手动发布关键赛程和比分。

### 页面白屏

处理：

- 回滚上一版本。
- 发布备用静态页面。
- DevOps 查看错误监控。

### 分享预览异常

处理：

- 检查 Open Graph 配置。
- 清理平台缓存。
- 使用默认分享图兜底。

### 预测结果明显异常

处理：

- 暂时隐藏异常预测模块。
- 回滚到上一版预测数据。
- 在数据说明中展示更新时间和修复状态。

## 9. 运营材料要求

需要提供：

- 首页推荐配置说明。
- 赛前内容运营日历。
- 赛中热门话题模板。
- 预测异常反馈模板。
- 版本发布记录模板。
